Map实现类介绍

Posted on 2019-06-27 Edited on 2023-03-28 In java基础

本文主要是对java中的Map实现类进行简单的对比了解(不包含java.util.concurrent包下的Map)。基于Jdk1.8.0_171

Map 接口存储一组键值对象，提供key（键）到value（值）的映射。

Map.Entry ：描述在一个Map中的一个元素（键/值对）。是一个Map的内部类。

Map实现类的框架图(Map实现类大多数都直接实现了Map接口(treeMap、EnumMap除外)，图中未体现)：

Map实现类简单对比：

  • HashMap
    • 无序
    • 非线程安全的
    • 底层使用拉链式散列算法，数据结构为数据+链表+红黑树。
    • 使用频率最高的Map实现类，关于HashMap更详细的源码分析，可以点击查看
  • LinkedHashMap
    • 继承自HashMap
    • 有序的Map：实现基于插入顺序排序
    • 底层维护了一个双向链表实现排序
  • TreeMap
    • 有序的Map：基于key的自然排序
    • 底层使用红黑树实现
  • EnumMap
    • 该类是专门针对枚举类设计的一个集合类，集合中的所有键必须是同一个枚举类的实例。
    • 当EnumMap创建后，会表现成一个数组array，这种表现方式是紧凑高效的。
    • EnumMap的顺序，由枚举类实例的定义顺序决定。
    • 不允许空键
    • 线程不安全
  • WeakHashMap
    • 只持有key的弱引用，所以当key被回收时，可自动删除对应key的那条记录。
  • IdentityHashMap
    • 一般Map实现中key的值相等（equals为true）就认为是同一个key，这个Map是以同一个对象（引用）作为判断依据，官方不建议使用。
  • HashTable
    • 类似于HashMap的功能，线程安全，老式实现，建议不要再使用！
