概述

ArrayList 是一种变长的集合类，基于定长数组实现

ArrayList 是无序的

ArrayList 中允许空值和重复元素

ArrayList 不是线程安全的

ArrayList 继承于AbstractList，实现了List、Cloneable、java.io.Serializable、RandomAccess 接口。类图结构如下：

ArrayList源码分析

构造函数

ArrayList一共有3个构造方法

增加

ArrayList有两个增加方法

add(e)：相当于直接在数组的尾部增加一个元素，步骤为：

  1. 判断是否需要扩容
  2. 在尾部增加元素

add(index,element)：在指定的位置增加元素，步骤为：

  1. 检查传入的index是否在正确范围内
  2. 判断是否需要扩容
  3. 在数组中移动index之后的数据
  4. 设置index处的元素为当前传入的元素
  5. size 加1

扩容

在上面增加元素的逻辑中，有一步是判断是否需要扩容，在此看一下具体是怎么扩容的，步骤：

  1. 对比是否需要扩容
  2. 计算扩容时使用的新的容量newCapacity
  3. 根据新的容量扩容数组

获取

ArrayList里面提供了一个根据索引获取元素的方法，步骤：

  1. 检查传入的索引
  2. 数组中取出元素

更新

ArrayList中提供了一个更新方法：set(index,element)

  1. 检查传入的index
  2. 得到旧的值，用于最后返回
  3. 设置传入的值到传入的位置

删除

ArrayList中提供了两个删除方法

remove(o)：在数组中删除传入的元素，步骤：

  1. 循环查找到当前元素的索引位置
  2. 根据1中索引位置删除元素

remove(index)：删除指定位置的元素，步骤

  1. 对传入的index进行检查
  2. 根据传入索引删除元素

其他方法

  • indexOf() 、lastIndexOf() 和contains()方法

    indexOf(o)：循环ArrayList判断其中是否包含传入元素，如果包含则返回所在位置索引，否则返回-1,因为存在循环所以这个方法的时间复杂度为O(n-x)

    lastIndexOf(o)：和indexOf(o)的区别就是遍历是倒叙的

    contains(o)：内部调用的是indexOf(o)方法，即时间复杂度也为O(n-x)

  • clear()、trimToSize() 和 ensureCapacity() 方法

    clear()：clear方法是把ArrayList里面的数组中的元素都设置为null，并设置size为0（注意此时数组的长度是不变的）

    trimToSize()：设置当前的数组的长度为size的值（即当前元素的数量）

    这个方法解决了数组长度较大，但是内部存储的数据其实较少，即此时十分浪费空间（既浪费内存），即可调用这个方法，设置数组的长度为当前的size

    ensureCapacity(minCapacity)：确定数组的容量,即根据传入的容量对数组进行扩容

    这个方法可解决的是一直放入数据多次扩容浪费性能的问题，此时如果能确定当前ArrayList中大约需要放入多少元素，可调用此方法进行一次性扩容

  • toArray()

    ArrayList中包含两个toArray()方法，均是返回当前ArrayList的数组形式，区别是一个返回泛型类型，一个返回Object[]

  • size()和isEmpty() 方法

    这两个方法都是直接使用的全局变量 size ，size是ArrayList中元素的数量(注意不是ArrayList的长度)

  • clone() 方法

    ArrayList中clone()方法内部也是调用System.arraycopy()方法，实现的是一层的深拷贝。

  • addAll() 方法

    ArrayList中有两个addAll()：

    addAll(c)：在数组的最后增加元素：先扩容，在数组最后增加

    addAll(index,c)：在指定的位置增加元素：检查index，扩容，在指定位置增加

  • retainAll() 和removeAll() 方法

    removeAll ：删除传入的集合中的元素

    retainAll ：求和传入集合的交集

总结

ArrayList 是一个常用且比较基础的集合类，它的结构简单，本质上是一个变长的数组，本文就分析到这里
