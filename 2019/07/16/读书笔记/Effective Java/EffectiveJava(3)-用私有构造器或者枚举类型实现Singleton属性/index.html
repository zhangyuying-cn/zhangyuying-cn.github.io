EffectiveJava(3)-用私有构造器或者枚举类型实现Singleton属性

Posted on 2019-07-16 Edited on 2023-03-28

第3条：用私有构造器或者枚举类型实现Singleton属性

Singleton指仅仅被实例化一次的类，即单例类

可使用三种方法：

1.公有域

2.静态工厂

关于序列化：

想要以上两种方式实现的Singleton类变成是可序列化的(Serializable)，仅仅在声明中加上“implements Serializable”是不够的，因为仅仅这样做，会导致每次反序列化一个序列化的实例时，都会创建一个新的实例。所以在声明中加上“implements Serializable”后，需要声明所有实例域都是瞬时的(transient)，并提供一个readResolve方法,即自己实现序列化

3.枚举

包含单个元素的枚举类型

枚举方式在功能上与公有域方法相近，但是它更加简洁，无偿的提供了序列化机制，绝对防止多次实例化，即使在面对复杂的序列化或者反射攻击的时候

单元素的枚举类型已经成为实现Singleton的最佳方法
