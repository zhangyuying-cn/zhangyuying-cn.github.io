Spring Boot

Posted on 2021-04-12 In Spring

Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can “just run”.

(Spring Boot使得创建独立的、生产级的、基于Spring的应用程序变得很容易，您可以“直接运行”。)

一、Spring Boot的理解

1.Spring Boot 是什么

提供了快速构建Spring Framework 应用的框架，即 Spring Framwork是Spring Boot的内核

2.Spring Boot 是如何简化Spring配置的

主要思想就是 约定优于配置：通过约定的方式来减少配置，从而提升开发效率，具体体现：

（1）默认的Maven结构，以及默认的打包方式

（2）默认的配置文件application.properties

（3）Starter的相关jar包依赖

（4）Web默认包含Spring-mvc以及内置tomcat容器

（5）部署到tomcat

二、简化配置的原理

1.Spring 注解驱动

由xml配置逐渐转变为注解驱动方式，通过注解实现Spring对Bean的管理

2.Spring Boot的自动装配

SpringBoot starter机制（自动装配）：SpringBoot中的starter是一种非常重要的机制，starter遵循“约定大于配置”的理念，将繁杂的配置和需要的Bean统一集成进starter，应用者只需要在maven中引入starter依赖，Spring Boot 就能：1)自动扫描到要加载的信息并启动相应的默认配置 ，2)自动通过classpath路径下的类发现需要的Bean，并注册进IOC容器

SpringBoot提供了针对日常企业应用研发各种场景的spring-boot-starter依赖模块

（1）自定义starter原理：

也是约定优于配置：Spring Boot启动时自动加载META-INF/spring.factories文件的配置Bean

  • 自定义的starter：在starter中的META-INF/spring.factories配置当前starter中使用@Configuration的配置类

（2）Spring Boot中的SPI机制实现

  • SPI思想

    • SPI的全名为Service Provider Interface.这个是针对厂商或者插件的。

    • SPI的思想：系统里抽象的各个模块，往往有很多不同的实现方案，比如日志模块的方案，xml解析模块、jdbc模块的方案等。面向的对象的设计里，我们一般推荐模块之间基于接口编程，模块之间不对实现类进行硬编码。一旦代码里涉及具体的实现类，就违反了可拔插的原则，如果需要替换一种实现，就需要修改代码。为了实现在模块装配的时候能不在程序里动态指明，这就需要一种服务发现机制。java spi就是提供这样的一个机制：为某个接口寻找服务实现的机制

  • 在Spring Boot的自动装配过程中，最终加载META-INF/spring.factories文件的配置，即是SPI思想的一种实现方式，

SQL的连表查询

Posted on 2021-01-01 Edited on 2021-01-07 In SQL

表的连接包括：内连接、左连接、右连接、外连接，交叉连接、左排除连接、右排除连接、非交集连接

  • 内连接查询 inner join

    该查询将返回两表满足匹配关系的记录。

  • 外连接

    外连接可分为：左连接、右连接、完全外连接

    • 左连接 left join left outer join

      获取左表中的所有记录，即使在右表没有对应匹配的记录。

      左外连接包含left join左表所有行，如果左表中某行在右表没有匹配，则结果中对应行右表的部分全部为空(NULL).

      注：此时我们不能说结果的行数等于左表数据的行数（只有表关系为一对一的时候相同）。

    • 右连接 right join

      右外连接包含right join右表所有行，如果左表中某行在右表没有匹配，则结果中对应左表的部分全部为空(NULL)。

    • 完全连接. Full join

  • 交叉连接 cross join
