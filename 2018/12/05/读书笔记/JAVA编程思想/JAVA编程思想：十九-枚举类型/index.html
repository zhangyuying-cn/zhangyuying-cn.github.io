JAVA编程思想：十九-枚举类型

Posted on 2018-12-05 Edited on 2023-03-28

关键字enum 可以将一组具名的值的有限集合创建为一种新的类型，而这些具名的值可以作为常规的程序组件使用，这是一种非常有用的功能

1.基本enum特性

enum的values()方法，可以遍历enum实例，alues()方法返回enum实例的数组，而且该数组严格保持其在enum中声明时的顺序

创建enum时，编译器会为你生成一个相关的类，这个类继承自ava.lang.Enum：

  • ordinal()方法返回一个int值：这是每个enum实例在声明时的次序，从0开始
  • 可以使用==来比较enum实例，编译器会自动为你提供equals()和hashcode()方法
  • Enum类实现了Comparable接口，所以它具有compareTO()方法，
  • 同时还实现了Serializable接口
  • name()返回enum实例声明时的名字
  • valueOf()是static方法，根据给定的名字返回相应的enum实例，如果不存在给定名字的实例，将会抛出异常

使用 静态导入（import static 路径）能够将enum带入当前的命名空间，所以无需再用enum类型来修饰enum实例

  • 注意：在定义enum的同一个文件中，这种技巧无法使用，如果是在默认包中定义enum,这种技巧也无法使用

2.向enum中添加新方法

除了不能继承自一个enum之外 ，基本可以将enum看做一个常规的类，我们可以先向enum中增加方法，enum中甚至可以有main()方法

java要求必须先定义enum实例，在定义实例之前定义任何方法和属性都会报错

enum中的构造方法和普通方法没有区别，不过默认为私有，我们只能在enum定义的内部使用其构造器创建enum实例

3.switch语句中的enum

switch中只能使用整数值（字符串时，使用的是hashcode）,而枚举实例天生就具备整数值的次序（并且可通过ordinal()获取）

所以可以在switch中使用enum

4.values()的神秘之处

values()是由编译器添加的static方法

由于values()是编译器插入到enum定义中的static方法，所以你将enum实例向上转型为Enum，那么values()方法就不可访问了，不过还可使用其他方式获取所有enum实例：Class中有一个getEnumConstans()方法。

getEnumConstants是Class上的方法，所以不是枚举类也可以使用：但是 此时此方法返回null。

因为所有的enum都继承自java.lang.Enum,所以**enum不能在继承其他的类**，但是可以实现一个或多个接口

8.使用EnumSet替代标志

java SE5引入EnumSet,EnumSet非常高效快速

使用EnumSet的优点是，他在说明一个二进制位是否存在时，具有更好地表达能力，并且无需担心性能

EnumSet中的元素必须来自一个enum

  • EnumSet.noneOf()
  • EnumSet.of()
  • EnumSet.allOf()
  • EnumSet.removeAll()
  • EnumSet.complementOf()

EnumSet的基础是long

9.使用EnumMap

EnumMap是一种特殊的map，要求其中的key必须来自一个enum，

由于enum本身的限制，所以EnumMap在内部可由数组实现，所以EnumMap的速度很快。

除了key值要求，其他操作与一般的map差不多

与EnumSet一样，enum实例定义时的次序决定了骑在EnumMap中的顺序

与常量方法相比：EnumMap有一个优点：允许改变值对象，而常量相关的方法，在编译期就被固定了
