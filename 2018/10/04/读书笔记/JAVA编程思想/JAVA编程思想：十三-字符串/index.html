JAVA编程思想：十三.字符串

Posted on 2018-10-04 Edited on 2023-03-28

可以证明，字符串操作是计算机程序中最常见的行为

1.不可变 String

String对象是不可变的。

String类中每一个看起来会修改String值的方法，实际上都是创建了一个全新的String对象，以包含修改后的字符串内容，而最初的String对象则丝毫未动

每当把String对象作为方法的参数时，都会复制一份引用，而该引用所指的对象其实一直待在单一的物理位置上，从未动过。

2.重载"+"与StringBuilder

不可变性会带来一定的效率问题。例如：String对象重载的+操作符，+可以用来连接两个String

注：重载的意思是：一个操作符在应用于特定的类时，被赋予了特殊的意义。用于String的+与+=是Java中仅有的两个重载过的操作符，而Java并不允许程序员重载任何操作符

想查看以上代码是如何工作的，可是用JDK自带工具：javap反编译以上代码。

生成字节码后可看出编译器自动引入了java.lang.StringBuilder类（我们在代码中没有使用），因为他更高效

注意：此时可能会觉得：可以随意使用String对象，反正编译器会为你自动优化性能。但是：若是在循环中是用+操作符来链接字符串，编译器会使得每经过一次循环，就会创建一个StringBuilder对象

因此：当为一个类编写toString()方法时，如果字符串操作比较简单，那就可以信赖编译器，他会为你合理地构造最终的字符串结果，但是如果在toString()方法中使用循环，那么要自己创建StringBuilder对象，用它构造最终的结果。

注意：使用StringBuilder时：append(a+":"+c) 时，编译器就会为你另外创建一个StringBuilder对象处理括号内的字符串操作

StringBuilder提供了丰富而全面的方法:

insert()、repleace()、substring()、reverse() 、delete()

最常用的：append()、toString()

Java5以后引入的StringBuilder,之前使用StringBuffer。StringBuffer是线程安全的，因此开销大些。

3.无意识的递归

当在toString()方法中使用this时，调用toString()会导致递归，示例：

因为编译器看到一个String对象（“test”）后面跟着一个+,而后的对象不是String，于是编译器试着将this转换成一个String,就是调用toString()方法，于是产生递归

如果真的想打印出对象的内存地址，应该调用Object.toString()方法，这才是负责此任务的方法，所以此处不应该是this而应该是super.toString()方法。

4.String上的操作

关于String对象具备的一些方法，可查看文档。

但是无论什么方法，当需要改变字符串内容时，String类的方法都会返回一个新的String对象，同时，如果内容没有发生改变，String的方法只是返回指向原对象的引用而已，这可以节约空间以及避免额外的开销。

5.格式化输出

格式修饰符：格式化输出时的占位符，他们不但说明了插入数据的位置，同时还说明了将插入类型的变量，以及如何对其格式化。例如：

  • %d ：表示整数；
  • %f ：浮点数（float或是double）,默认小数点后六位（十进制）
  • %s ：表示字符串
  • %c ：Unicode字符
  • %b ：Boolean值
  • %e：浮点数（科学计数）
  • %x ：整数（十六进制）
  • %h ：散列码（十六进制）
  • %%：字符 “%”

System.out.format() === System.out.printf() 可用于PrintStream或PrintWriter

Formatter ：Java中，所有格式化功能都由java.util.Formatter类处理。

当创建一个Formatter对象的时候，需要向其构造器传递一些信息，告诉他最终的结果将向哪里输出

6.正则表达式

（1）正则表达式是一种强大而灵活地文本处理工具。

正则表达式提供了一种完全通用的方式，能够解决各种字符串处理相关的问题：匹配、选择、编辑验证。

注：java 中的正则普遍是多一个\,换种说法就是java双斜杠（\\）表示插入一个正则表示的斜线

例如：\d ：一位数字，在java的正则中表示为：\\d；

（2）**String类自带的使用正则表达式的方法**：

  • split()：将字符串从正则表达式的地方分隔。还有一个重载版本，可以限制字符串分割的次数
  • 替换类方法：
    • replaceFirst()：只替换正则表达式第一个匹配的字符串
    • replaceAll()：替换所有匹配的地方
  • matches()：是否和传入的正则匹配

如果正则不是只使用一次的话，非String对象的正则表达式明显具备更佳的性能

（3）正则表达式的完整构造字列表，参考java.util.regex包中的Pattern类

（4）量词

量词描述了一个模式吸入文本的方式

  • 贪婪型：贪婪表达式会为所有可能的模式发现尽可能多的匹配
  • 勉强型：用问号来指定，这个量词匹配满足模式所需的最小字符数
  • 占有性：目前这种类型的量词只在java中才可用，较为复杂。

（6）Pattern 和 Matcher

Pattern.compile() :编译正则表达式，生成Pattern对象。Pattern对象表示编译后的正则表达式

Pattern的对象的 matcher()：会生成一个Matcher对象.

Matcher有许多功能可用：

  • find()：在CharSequense中查找多个匹配,若是传入参数，则从传入的位置开始向后查找；
    • matches()：整个输入都匹配正则表达式时成功
    • lookingAt()：只要输入的第一部分匹配就会成功
  • start()：先前匹配的起始位置索引
  • end()：所匹配的最后字符索引加一的值
  • Groups:
    • 正则表达式：A(B(C))D：组0是ABCD;组1是BC;组2是C.
    • group()：前一次匹配操作的第0组
    • group(int)：返回再前一次匹配操作期间指定的组号；没有则返回null
    • groupCount()：返回该匹配器的模式中的分组数目
    • start()：前一次匹配操作中寻找到的组的起始索引
    • end()：前一次匹配操作中寻找到的最后一个字符索引加一的值

（8）替换操作

  • replaceFirst(String replacement)：以参数字符串replacement替换掉第一个匹配成功的部分

  • replaceAll(String replacement))：以参数字符串replacement替换所有匹配成功的部分

  • appendRepalcement(StringBuffer sbuf, String replacement)：执行渐进式替换（而不是像replceFirst()和replaceAll()那样只替换第一个匹配或是全部匹配）它允许你调用其他方法来生成或处理replacement,使你能够用编程的方式将目标分割成组

  • appendTail(String sbuf)：在执行了一次或多次appendRepalcement()之后，调用此法可以将输入字符串余下的部分复制到sbuf中

（9）reset()

可以将现有的Matcher对象应用于一个新的字符序列

使用不带参数的reset()方法，可以将Matcher对象重新设置到当前字符序列的其实位置

7.扫描输入

javaSE5 新增Scanner类，减少扫描输入的负担

Scanner的构造器可以接受任何类型的输入参数：包括File、InputStream、String、Readable(接口，表示“具有read()方法的某种东西”，例如BufferedReader就归属于这类)

Scanner的所有输入、分词以及翻译的操作都隐藏在不同类型的next()方法中

普通的next()方法返回下一个String
所有的基本类型（除了char之外），都有对应的next()方法（其中还包括bigDecimal和BigInteger）
所有的next()方法，只有在找到一个完整的分词后才会返回
Scanner还有相应的 hasNext()方法，用以判断下一个输入的分词是否是所需的类型

Scanner定界符：

Scanner根据空白字符对输入进行分词，但是你可以用正则表达式指定自己所需的定界符

用正则表达式扫描：

next()方法配合正则使用时，将找出下一个匹配该模式的输入部分。调用match()方法就可获得匹配的结果

注：再配合正则扫描时，仅仅针对下一个分词进行匹配，如果你的正则表达式中含有定界符，那永远不可能匹配成功

8.StringTokenizer

在jdk1.4引入正则和jdk1.5引入scanner之前，分隔字符串的唯一方法是使用StringTokenizer来分词。

现在StringTokenizer已经被废弃了

9.总结

Java现在对字符串操作的支持已经很完善了，不过，在使用时要在细节上注意效率问题：例如恰当的使用StringBuilder等
